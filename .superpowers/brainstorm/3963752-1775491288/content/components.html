Section 3: Frontend Component Architecture

How the React components fit together

Component Tree
// New components (features/call-map/)
<CallMapProvider> ← wraps App, listens for location_detected WS events
<CallMapFloater> ← floating pop-over, renders anywhere
<MiniMap> ← compact MapLibre GL, ~380×200px
<ZoneIndicator> ← "✓ Core" / "⚠ Extended" / "✗ Outside"
<DriveTimeChip> ← "🚗 ~12 min from base"
<LocationSource> ← shows where detection came from
// Enhanced existing component
<SoftPhone> ← existing, gets new map panel
<CallMapPanel> ← full side panel, replaces floater when open
<FullMap> ← larger MapLibre GL with zone polygons
<ZoneLegend> ← core (red) + extended (amber dashed)
<CustomerInfoCard> ← if matched, show customer details
<TranscriptHighlight> ← quoted text that triggered detection
// State management
<callMapStore> ← Zustand: location, zone, isVisible, isExpanded
Data Flow: Location Detection → Map Update
🎙️
WebSocket
location_detected event arrives
→
🏪
callMapStore
Updates location, zone, confidence
→
🗺️
CallMapFloater
Auto-shows, flies map to pin
→
👆
User Action
Dismiss, or "Open Full View"
KEY BEHAVIORS
📍 New location replaces old pin (smooth fly-to animation)
🔇 Floater auto-hides when call ends (5s delay)
🎯 Higher confidence location replaces lower one
📌 "Open Full View" navigates to /phone with map panel

Zone Display Behavior

✓ Core Service Area

Green badge + solid polygon on map. Confident "yes we service there."

⚠ Extended Service Area

Amber badge + dashed polygon. "We can get there — ~45 min drive."

✗ Outside Service Area

Red badge, no polygon highlight. "That's outside our current coverage."

These aren't choices — just showing how each zone state would display. Does this component structure and behavior look right?
